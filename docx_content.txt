SMARTFOOD OPS 360
PRD — Versăo Simplificada
Engenharia de Alimentos  ·  Etiquetas  ·  Insumos e Fornecedores
Visăo Geral
O SmartFood Ops 360 começa pelos 3 pilares que realmente impactam a margem de uma cozinha industrial de salgados: saber o custo real de cada receita, imprimir a etiqueta certa no lote certo, e conhecer o rendimento real de cada marca de insumo comprada.

Estrutura de Pastas do Projeto
1.1  Objetivo
Permitir que o Chef monte a receita de qualquer salgado dividida em seçőes livres (Massa, Recheio, Cobertura...), com cálculo automático de custo real usando Fator de Correçăo e Fator de Cocçăo por ingrediente.

2.1  Objetivo
Editor visual de etiquetas térmicas que mostra em tempo real como a etiqueta vai ficar antes de enviar para a impressora. Compatível com Zebra (ZPL) e Elgin/Argox (TSPL). Campos preenchidos automaticamente a partir do lote de produçăo.
2.2  Parâmetros da Etiqueta
2.3  Tela do Editor de Etiqueta
2.4  Preview HTML da Etiqueta
O preview é um div HTML estilizado com Tailwind que simula a aparęncia física da etiqueta. Fundo branco, borda preta, fonte monoespaçada, proporçăo em escala. O QR Code é um SVG gerado via biblioteca qrcode Python ou placeholder visual.
2.5  Modelo de Dados — Módulo 2
3.1  Objetivo
Separar o insumo genérico (ex: 'Peito de Frango') de suas marcas reais (Seara, Frisa, Aurora). Cada marca tem seu próprio Fator de Correçăo, percentual de rendimento e preço de compra. Assim o custo na Ficha Técnica usa o rendimento real da marca que vocę compra.
